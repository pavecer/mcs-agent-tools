# o1 — System Instruction Best Practices

o1 is OpenAI's first reasoning model. It performs internal chain-of-thought reasoning before responding. This fundamentally changes how system instructions should be written: **focus on goals and constraints, not on reasoning process.**

---

## Key Difference: Goal-Oriented, Not Process-Oriented

o1 reasons through problems automatically. Do not instruct it on *how* to think — focus on *what* you want as output.

| ❌ Redundant for o1 | ✅ Effective for o1 |
|---|---|
| "Think step by step" | "Provide a structured analysis" |
| "First consider X, then reason about Y" | "Your output must address X and Y with evidence" |
| "Reason carefully before responding" | "Be precise and cite all sources" |
| "Chain of thought:" | "Format your response as follows:" |

---

## 1. Define Output Requirements, Not Reasoning Steps

Since o1 handles its own reasoning, instructions should focus on **what the output must contain and how it should look**.

```
For each policy question, provide:
1. The applicable rule or clause (quoted verbatim from source).
2. A plain-language interpretation (2–3 sentences).
3. Any exceptions or caveats explicitly stated in the source.
If the source does not address the question, state: "Not found in available sources."
```

---

## 2. Set Constraints, Not Procedures

```
Only use information from the provided documents.
Do not infer or extrapolate beyond what is explicitly stated.
If the answer cannot be found in the sources, state that explicitly — do not guess.
```

---

## 3. Persona and Scope

```
You are a compliance assistant for [Your Organisation].
You answer questions strictly within the domain of [Your Organisation]'s internal compliance policies.
Do not advise on external regulations unless they are cited in the provided documents.
```

---

## 4. Avoid Redundant Reasoning Prompts

The following add no value with o1 and consume token budget:

- "Let's think through this carefully"
- "Consider all factors before responding"
- "Chain of thought:"
- "Step 1: analyse the question"
- "Reason step by step"

o1's internal reasoning is already more thorough than prompted reasoning — these instructions are noise.

---

## 5. Grounding Rules

Even for a reasoning model, grounding rules are essential when working with private data.

```
Answer exclusively from the provided search results or documents.
Do not use general training knowledge for factual claims.
Cite the source name for every substantive statement.
```

---

## 6. Recommended Lengths

| Use Case | Target Length |
|---|---|
| Focused expert assistant | 100–500 chars |
| Enterprise knowledge agent | 300–3,000 chars |
| Complex multi-constraint agent | 500–8,000 chars |
| Maximum recommended | 10,000 chars |

---

## 7. Common Mistakes

- **Over-specifying the reasoning process:** o1's strength is internal reasoning — trust it; directing it step-by-step wastes budget and may interfere.
- **Too long for scope:** o1 works best when instructions are tight and goal-focused rather than exhaustive.
- **Mixing format with reasoning instructions:** Define output format separately from content requirements.
- **Identical instructions to GPT-4.x:** o1 requires a different prompting strategy; reusing older prompts unchanged is suboptimal.
